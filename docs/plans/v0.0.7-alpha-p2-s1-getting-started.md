## Claude Code Quick Start

**Prerequisites**:
- Claude Code plugin installed ([get it here](https://marketplace.visualstudio.com/...))
- Git installed (optional but recommended)

**Steps**:

1. **Initialize knowledge graph** (30 seconds)
   ```bash
   /knowledge:init
   ```
   Claude will ask:
   - Project name? (use your current project name)
   - Enable git tracking? (recommended: yes)

2. **Verify setup** (10 seconds)
   ```bash
   /knowledge:status
   ```
   Should show: "Knowledge Graph: [your-project], 0 lessons"

3. **Capture your first lesson** (3-4 minutes)
   ```bash
   /knowledge:capture-lesson
   ```
   Think of a recent problem you solved. Claude will guide you through documenting it.

4. **Verify it worked** (10 seconds)
   ```bash
   /knowledge:status
   ```
   Should show: "1 lesson"

**Done!** You've captured your first lesson.

**Next steps**:
- Read [Essential Commands](COMMAND-GUIDE.md#essential-commands)
- Browse [examples](../core/examples/) to see what's possible
- Set up [sanitization](CONFIGURATION.md#sanitization) before sharing code

## AI Assistant Setup

**Prerequisites**:
- AI assistant installed (Cursor, Continue, Aider, etc.)
- Git installed (recommended)
- Terminal access

**Steps**:

1. **Install core system** (2 minutes)
   ```bash
   # Clone or download the core system
   cd your-project
   mkdir -p docs

   # Copy core templates and scripts
   cp -r /path/to/knowledge-graph/core docs/kg-core
   ```

2. **Set up directory structure** (1 minute)
   ```bash
   cd docs
   mkdir -p lessons-learned decisions knowledge sessions
   cp -r kg-core/templates/. templates/
   ```

3. **Configure your AI assistant** (2 minutes)

   Add this to your AI assistant's instructions:
   ```
   When the user asks to document a lesson:
   1. Copy docs/templates/lessons-learned/lesson-template.md
   2. Fill in [MANUAL] fields based on conversation
   3. Save to docs/lessons-learned/[category]/[Title].md
   4. Leave [AUTO] fields for user to fill
   ```

4. **Create your first lesson** (5 minutes)

   Ask your AI assistant:
   "Help me document a lesson I just learned about [your topic]"

## Manual Setup

**Prerequisites**:
- Git installed (optional)
- Text editor (vim, VSCode, etc.)

**Steps**:

1. **Download templates** (1 minute)
   ```bash
   # Clone the core system (or download from GitHub releases)
   git clone https://github.com/technomensch/knowledge-graph.git /tmp/kg-source

   # Copy just the core system to your project
   cp -r /tmp/kg-source/core your-project/docs/kg-core
   ```

2. **Create directory structure** (1 minute)
   ```bash
   mkdir -p lessons-learned/{architecture,process,patterns,debugging}
   mkdir -p decisions knowledge sessions
   ```

3. **Create your first lesson** (10 minutes)

   **Copy template**:
   ```bash
   cp templates/lessons-learned/lesson-template.md \
      lessons-learned/debugging/My_First_Lesson.md
   ```

   **Open in editor**:
   ```bash
   vim lessons-learned/debugging/My_First_Lesson.md
   ```

   **Fill in [MANUAL] fields** (see field markers in template):
   - `title`: "Lesson: [What you learned]"
   - `category`: Pick one (architecture/process/patterns/debugging)
   - `tags`: [relevant, keywords]

   **Write content** in the body sections:
   - What was the problem?
   - What was the root cause?
   - How did you solve it?
   - How to prevent it?

4. **Save and commit** (if using git)
   ```bash
   git add lessons-learned/debugging/My_First_Lesson.md
   git commit -m "docs: add first lesson"
   ```

## Conceptual Overview

**What is this?**

A knowledge graph is a structured way to capture and organize what you learn while working.

**Why use it?**

- **Remember solutions**: Solved a tricky bug? Document it. Next time (or when a teammate hits it), search and find the fix in 30 seconds.
- **Share knowledge**: New team member? Point them to your lessons. They get months of context in hours.
- **Learn patterns**: After 10 database lessons, patterns emerge. The system extracts them automatically.
- **Feed your AI**: Best lessons sync to your AI assistant's memory, so it "remembers" your patterns.

**How does it work?**

1. **Capture** lessons as you solve problems
2. **Organize** into categories (architecture, debugging, etc.)
3. **Search** when you need to recall something
4. **Sync** patterns to AI memory (optional)

**What makes it a "graph"?**

- Lessons link to decisions ("Why we chose X")
- Decisions link to knowledge ("How X works")
- Git metadata links to code ("Where we implemented X")
- Everything searchable, everything connected

**Is this just fancy note-taking?**

Kind of! But with structure that enables:
- ‚úÖ Fast search (find lessons by date, tag, category)
- ‚úÖ Consistency (templates ensure nothing missed)
- ‚úÖ Automation (AI can read and write entries)
- ‚úÖ Team sharing (sanitization built-in)

## Troubleshooting

### "I installed but commands don't work"
- **For Claude Code**: Restart Claude Code, verify plugin installed
- **For other AI**: Check AI instructions include template paths

### "Which category should I use?"
- **architecture**: System design, how components fit together
- **process**: Workflow improvements, tools, procedures
- **patterns**: Reusable solutions, best practices
- **debugging**: Bug fixes, troubleshooting

### "Do I need git?"
- **Recommended** but not required
- With git: Auto-tracks branch/commit/PR metadata
- Without git: Still works, just no automatic linking to code
