## Key Concepts

- **Knowledge Graph**: Structured collection of lessons, decisions, patterns
- **YAML Frontmatter**: Metadata at top of files (title, date, tags, etc.)
- **Git Metadata**: Automatic tracking of branch, commit, PR, issue
- **MEMORY.md**: Persistent context file synced to Claude's prompt
- **Sanitization**: Removing sensitive data before sharing
- **Meta-Issue**: Multi-attempt problem tracking system

## Common Workflows

### First Time Setup (5 minutes)
1. `/knowledge:init`
2. `/knowledge:capture-lesson` (document your first learning)
3. `/knowledge:status` (verify it worked)

### Daily Use (10 minutes)
1. Solve problem or learn something
2. `/knowledge:capture-lesson` (document it)
3. `/knowledge:update-graph` (sync to KG)

### Before Sharing Code (2 minutes)
1. `/knowledge:check-sensitive` (scan for secrets)
2. Review findings
3. Remove sensitive data before git push

## Quick Tips

- `/knowledge:recall` searches everything (lessons, decisions, knowledge)
