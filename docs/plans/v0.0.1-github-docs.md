**Target Release:** v0.0.1-github-docs
**Status:** In Progress
**Branch:** `v0.0.1-github-docs`

---

## Context

Alpha testers reported the homepage doesn't clearly answer three questions: what the project is, why it should matter to them, and how to use it. The current page leads with a self-deprecating "personal learning project" info box, followed immediately by a warning admonition â€” two signals that discourage engagement before the value proposition appears. The content itself is strong; the ordering is wrong, and audience/use-case context is missing entirely.

**Goal:** Restructure `docs/index.md` for a value-first layout that adds audience identification and concrete use cases. Existing language is preserved (with explicitly noted edits where needed). No other files change.

---

## Decision: Pure Markdown Only (no HTML overrides)

Material for MkDocs has a `home.html` override template for hero sections, but it adds complexity (new directory, Jinja2 inheritance, mkdocs.yml structural change) not justified here. The issue is **information architecture**, not visual styling. The existing CSS and grid card system already produce a polished result.

**Files changed:** `docs/index.md` only
**No changes to:** `mkdocs.yml`, `extra.css`, any other doc
